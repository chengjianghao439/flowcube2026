# FlowCube v0.3.19

## 摘要

桌面端 **二级确认与提示** 统一为 **系统原生 `dialog.showMessageBox`**（自动更新、退出应用、未保存离开、`confirmAction`）；移除渲染层 `DesktopMessageBoxBridge` / `DesktopQuitDialog`。货架本机打印在 **非桌面环境** 下给出明确提示；打印任务 `printerName` 在库内为空时用 `printerCode` 兜底，便于本机 RAW 队列名。

## 桌面端（Electron）

- **`updateCheck.js`**：更新流程提示不再经 IPC 走 React `AppDialog`，直接 **`dialog.showMessageBox`**。
- **`main.js`**：窗口关闭用系统 **退出确认**；新增 **`flowcube:show-message-box`** IPC。
- **`preload.js`**：暴露 **`showMessageBox`**；移除 `onCloseRequest`、渲染层 MessageBox 桥接相关 API；保留 **`acceptClose`**（菜单等场景）。
- **`DesktopQuitUnloadBridge`**：主进程确认退出后派发 `flowcube-quit-confirmed`，配合 **`beforeunload` 闸门**，避免与脏页保护双重提示。

## 前端

- **`dirtyGuardStore.showConfirm` / `GlobalConfirmDialog`**：在 **`VITE_ELECTRON=1`** 且存在桥接时走原生框；否则仍为 **`ConfirmDialog`**（PDA / 浏览器）。
- **`desktopLocalPrint.ts`**：错误信息格式化更稳健；非 `Error` 对象时避免 `[object Object]`。
- **`pages/racks/index.tsx`**：本机打印为 `skipped` 且桌面桥接不可用时 **`toast.warning`** 说明需使用桌面安装包。

## 后端

- **`print-jobs.service.js`**（`fmt`）：**`printerName`** 在 **`printer_name` 为空** 时用 **`printer_code`** 兜底，满足本机 RAW 对队列名的需要。

## 升级说明

1. **后端**：若使用货架/本机打印队列名兜底，部署并重启。
2. **桌面**：CI 推 **`v0.3.19`** tag 后等待 **Build Desktop Installer** 完成，自 GitHub Release 下载安装包。

## 相对 v0.3.18

原生化桌面提示与退出确认；货架打印在非桌面场景的引导提示；打印任务格式化与本地打印错误提示改进。
