# v0.3.40

## 后端

- **字符集**：连接池在每条连接上执行 `SET NAMES utf8mb4 COLLATE utf8mb4_unicode_ci`，与会话、表结构 utf8mb4 一致，减轻用户姓名等中文乱码。
- **500 排障**：`ER_NO_SUCH_TABLE` / `ER_BAD_FIELD_ERROR` 返回可读说明；环境变量 **`APP_EXPOSE_ERRORS=1`** 时向客户端暴露详细异常（仅排障期使用）。
- **自动更新**：`/api/app-update/latest` 在版本与本地 `latest.json` 一致时优先使用其中的 **`filename`**，与 `GET /downloads/` 下实际 exe 名一致，避免 404。
- **JSON**：延续 `app.set('json charset', 'utf-8')`。

## 运维

- **`scripts/dump-db-for-backup.sh`** / **`scripts/restore-db-from-dump.sh`**：本机 mysqldump 与 Docker MySQL 全库恢复（utf8mb4）。

## 桌面端（构建）

- **artifactName**：`Jixu-Flow-Setup-${version}.${ext}`，Release 与 `latest.json` 文件名统一为 ASCII，避免特殊命名导致的路径问题。

## 发版后部署

- 将 **`backend/downloads/latest.json`** 与 **`Jixu-Flow-Setup-0.3.40.exe`** 放到线上 `downloads`（或与 CI 产物一致）。
- 业务姓名仍乱码时，优先检查 **数据库备份导入是否使用 `--default-character-set=utf8mb4`** 及表 collation。
