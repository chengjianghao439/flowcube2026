# FlowCube v0.3.37

## 摘要

**商品标签接入正式打印链路**：商品管理页的“标签”按钮不再使用浏览器内临时 HTML 打印，而是接入后端打印任务体系，使用 **商品标签默认模板（type 8）** 与 **商品标签打印机绑定（product_label）** 统一出纸。

**打印代码收口**：移除前一轮面向特定 TSPL 打印机的兼容调试层，保留通用的 `ZPL / TSPL` RAW 打印主链路，减少打印机管理和桌面端的旁路逻辑。

## 主要文件

- `backend/src/modules/products/products.routes.js`
- `backend/src/modules/products/products.controller.js`
- `backend/src/modules/products/products.service.js`
- `backend/src/modules/print-jobs/print-jobs.service.js`
- `frontend/src/api/products.ts`
- `frontend/src/pages/products/index.tsx`

## 升级说明

- 若要从商品管理直接打印，请先在“打印机管理”中为 **商品标签** 绑定一台标签机。
- 商品标签版式取 **打印模板 type 8** 的默认模板；未配置时会使用内置兜底标签布局。

把商品标签真正并入了模板和打印机管理体系，同时回收上一轮特定机型调试代码。
