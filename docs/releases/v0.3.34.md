# FlowCube v0.3.34

## 摘要

**佳博 / TSC 本机 RAW**：当 **Windows 测试页可打**、但 FlowCube 任务 **队列显示成功却不出纸** 时，多与 **TSPL 中 `CODEPAGE` 行或固件兼容性** 有关。本版 **默认不再在服务端生成的 TSPL 里插入 `CODEPAGE`**，由 **Windows 桌面端** 继续默认以 **GB18030** 下发中文；需在指令里显式声明代码页的，可在 **打印模板 body** 中自行书写。

**桌面假脱机**：WinSpool 作业显示名由原来的「FlowCube ZPL」改为 **「FlowCube RAW」**，避免误以为发送的是 ZPL 协议。可选环境变量 **`FLOWCUBE_TSPL_OMIT_CODEPAGE=1`**：在下发前 **删除脚本中所有 `CODEPAGE` 行**（兼容不认该指令的固件）。

## 相对 v0.3.33

- 撤回「内置 TSPL 一律带 CODEPAGE 936」的策略，降低队列假成功、物理不出纸的概率。
- 与 `docs/RELEASE.md` 中 TSPL 编码说明一致。

## 升级说明

- **后端**：须部署，使新打印任务生成 **无默认 CODEPAGE** 的 TSPL。
- **桌面端**：须安装本版以包含队列标题与 `OMIT_CODEPAGE` 逻辑。
- **`GET /downloads/latest.json`**：与仓库 `backend/downloads/latest.json` 对齐。
