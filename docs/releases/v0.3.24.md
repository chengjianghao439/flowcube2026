# FlowCube v0.3.24

## 摘要

**ZPL 标签打印模板**：打印模板类型扩展为 **货架 / 散件容器 / 物流箱贴 / 商品 / 库存**（type 5–9），支持在设置中 **编辑 ZPL 正文** 与 **`{{占位符}}`**；迁移时 **种入默认 ZPL**（含 `^CI28`）。**打印任务**在存在对应类型的 **默认模板** 时优先使用模板内容，否则回退内置 `build*Zpl`。

**确认对话框**：`ConfirmDialog` / `GlobalConfirmDialog` 等小幅调整，与业务页（如拣货波次）配合。

## 后端

- **`labelZplTemplate.js`**（新）：`applyZplTemplate`、`getDefaultZplBody`（读取 `print_templates` 中 `is_default=1` 且 `format=zpl` 的布局）。
- **`print-jobs.service.js`**：货架 / 容器 / 箱贴入队时套用默认 ZPL 模板；内置 ZPL 增加 `^CI28`。
- **`print-templates.service.js`**：类型 5–9 校验与 `TYPE_NAME`；ZPL 与画布类布局校验。
- **`migrate.js`**：`seedDefaultPrintTemplates` 增加 type 5–9 默认 ZPL 种子。

## 前端

- **`settings/print-templates`**：类型 5–9 使用 **ZPL 文本编辑**（占位符说明、恢复默认、示例替换预览）；列表 **类型** 展示 `typeName`。
- **`constants/labelZplDefaults.ts`**：与后端种子一致的默认 ZPL 与占位符说明。
- **`print-template.ts`**：`TemplateLayout` 联合类型；`isZplTemplateLayout`。
- **`TemplateRenderer`**：ZPL 布局时给出说明，避免误用画布渲染。

## 升级说明

1. **后端**：部署并 **执行迁移**（或确保 `print_templates` 已种入 5–9 默认行），否则打印仍走代码内兜底 ZPL。
2. **桌面**：安装 CI 构建的新安装包（见 GitHub Release）；前端随桌面包更新。
3. **`backend/downloads/latest.json`**：发布安装包后把 `version` / `filename` / `notes` 指向本版本（若使用在线更新）。

## 相对 v0.3.23

ZPL 标签可配置、默认模板与打印链路对齐；确认框与部分页面交互优化。
