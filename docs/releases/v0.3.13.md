# FlowCube v0.3.13

## 摘要

本版本改进**货架补打条码**与打印链路的可观测性：接口返回 `dispatchHint`，前端按状态区分成功/警告；打印客户端支持 **ZPL 经 CUPS 原始队列**（macOS/Linux），并加固 TCP 发送。

## 后端

- **`print-jobs`**：新增 `getDispatchHintForJob`，根据任务状态与在线 SSE 客户端判断 `dispatched` / `no_print_client` / `queued_concurrency` / `done` / `failed` 等。
- **`POST /api/racks/:id/print-label`**：响应 `data` 增加 `dispatchHint`；`message` 随状态变化（已下发 / 未连接打印客户端 / 并发排队 / 默认入队说明）。

## 打印客户端（`frontend/print-client.js`）

- **ZPL**：仍优先 **TCP 网口**（`host` / `--zpl-host`，默认 9100）；若无网口配置，在 **macOS/Linux** 上可使用路由表中的 **`lp`** 或 **`--local-printer`**（CUPS 队列名），通过 **`lp -d <队列> -o raw`** 发送。
- **TCP**：`write` 完成后再 `end()`，并避免 `close`/`error` 重复 settle。

## 前端

- **货架列表**：打印结果根据 `dispatchHint` 展示 **warning**（无在线客户端、并发排队）或 **success**（已下发至工作站等）。

## 升级与运维说明

1. **Windows 拉代码**：在仓库根目录执行 `git pull origin main`，与 GitHub `main` 对齐；桌面安装包需按既有流程重新 `dist:win` 或从 Release 下载新版本。
2. **仅 CUPS、无斑马 IP**：在打印工作站为 macOS/Linux 时，可为对应逻辑打印机配置 **`lp` 队列名** 或 **`--local-printer`**，无需单独填 `zpl-host`（ZPL 走 raw 队列）。
3. 发布 **GitHub Release** 时：使用 tag **`v0.3.13`**，说明可摘抄本文「摘要 + 各节要点」。
