# v0.3.41

## ERP Web

- **采购 / 收货分离**：顶栏「采购」菜单包含 **采购订单**（`/purchase`）与 **收货订单**（`/inbound-tasks`）；原「仓库任务」内的「入库任务」入口已移除，避免与采购业务混在同一组。
- **权限**：新增 **`page:inbound`**（收货订单列表与详情）。默认赋予仓库管理员、采购员、只读等角色（见 `frontend/src/lib/permissions.ts`）；权限管理页可勾选「收货订单」。
- **文案**：采购列表/详情、收货列表/详情、库存提示、PDA 收货相关用语与「收货订单」一致；自采购单创建任务时按钮为「创建收货订单」。

## 数据库

- **`057_add_page_inbound_permission.sql`**：向 `sys_role_permissions` 为角色 2、3、5 插入 `page:inbound`（`INSERT IGNORE`）。

## 桌面端（构建）

- 版本号 **`0.3.41`**（与 tag **`v0.3.41`**、`desktop/package.json` 一致）。
- 安装包文件名：`Jixu-Flow-Setup-0.3.41.exe`。

## 发版后部署

1. 运行迁移，确保 **`057_add_page_inbound_permission.sql`** 已执行。
2. 将 **`backend/downloads/latest.json`** 与 **`Jixu-Flow-Setup-0.3.41.exe`** 放到线上 `downloads`（或与 CI/GitHub Release 产物一致）。
3. 确认 **`GET .../downloads/latest.json`** 中 `version`、`filename` 与本发行一致。

## 发布方式

在 `main` 已推送且工作区干净的前提下：`npm run release:tag-desktop` → 推送 **`v0.3.41`** → GitHub Actions **Build Desktop Installer** 构建 Windows 安装包并挂到 Release。
