# FlowCube ERP 桌面端 v0.3.7 更新说明

对应 Git 标签：`v0.3.7`（与 `desktop/package.json`、`frontend/package.json` 版本一致）。

## 更新内容（本版）

- **交互统一**：移除页面中的原生 `alert` / `confirm`，改为 `toast` 与通用 `ConfirmDialog`，避免阻塞式系统弹窗。
- **表单与筛选**：将多处原生 `<select>` 替换为 Radix UI `Select`（`@/components/ui/select`），与现有入库任务、看板等页面风格一致。
- **涉及范围**：客户绑定价格表、入库任务取消、PDA 登录与拣货、打印机管理、库存/总览/盘点、采购与销售相关表单与列表、退货与调拨、仓库任务（含优先级与状态筛选）、库位与货架表单、价格表、账款、操作日志、打印模板与租户设置等。

## 自动更新测试提示

1. 安装 **低于 0.3.7** 的旧版桌面端（例如 v0.3.6）。
2. 确保后端可访问，且 `GET /downloads/latest.json`（或你们配置的更新接口）返回本版本的 `version` 与 `filename`；若使用 GitHub Release 直链，客户端会按逻辑尝试同域 `/downloads/` 文件名。
3. 在客户端触发「检查更新」或等待定时检查；应提示新版本并下载 `FlowCube-ERP-Setup-0.3.7.exe`（与 CI 重命名后的安装包名一致）。

## 构建与发布

- GitHub Actions：**Build Desktop Installer**，由推送 `v0.3.7` 标签触发，生成 Windows NSIS 安装包并创建/更新 GitHub Release。
- 若配置了 `SSH_PRIVATE_KEY` 等 Secret，工作流还会将 exe 上传到服务器的 `downloads` 目录，与 `latest.json` 配合提供内网下载。
