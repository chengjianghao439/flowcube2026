# FlowCube v0.3.32

## 摘要

**本机 RAW（TSPL）**：默认**不再**强制把 TSPL 换成 CRLF（恢复与 v0.3.30 一致的行为，避免部分佳博/TSC 在强制 CRLF 后反而不执行）。若某台机器**确实**需要 CRLF，可在系统中设置环境变量 **`FLOWCUBE_TSPL_CRLF=1`** 后重启桌面端（见 `docs/RELEASE.md`）。

**本机打印校验**：Web 在送 RAW 前校验接口声明的 **ZPL/TSPL** 与**实际内容**是否一致，不一致时给出明确错误提示，便于排除「打印机管理里指令集未生效」等问题。

**货架管理 UX**：若当前为**普通浏览器**或**桌面端未加载预加载桥接**，页顶展示说明，减少「Windows 打印队列为空却以为配置错了」的误判；打印成功提示会带出**目标打印机名称**；`docs/RELEASE.md` 补充 TSPL 换行说明。

## 主要变更

- `desktop/lib/localPrint.js`：TSPL CRLF 改为可选（`FLOWCUBE_TSPL_CRLF`）；日志含 `TSPL_EOL=native|CRLF`。
- `frontend/src/lib/desktopLocalPrint.ts`：ZPL/TSPL 与 `contentType` 交叉校验；`getLocalPrintEnvironmentKind()`。
- `frontend/src/pages/racks/index.tsx`：环境提示条、文案与 `skipped_no_desktop` 说明。
- `docs/RELEASE.md`：本机 TSPL 换行环境变量。

## 升级说明

- **桌面端**：安装本版以带上述 `localPrint` 逻辑。
- **Web**：部署前端以带上货架页与校验逻辑。
- 使用 **`GET /downloads/latest.json`** 时，请将线上文件与仓库 `backend/downloads/latest.json` 对齐。

## 相对 v0.3.31

修正部分机型在 v0.3.31 因默认 CRLF 导致不出纸的风险；强化「必须用桌面客户端才会进 Windows 队列」的可发现性。
