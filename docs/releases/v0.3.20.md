# FlowCube v0.3.20

## 摘要

强化 **桌面端 ZPL 本机 RAW 出纸** 的稳定性：RAW 前校验 **打印机是否存在于本机**（Windows `Get-Printer` / Unix `lpstat`）、校验 **ZPL 含 ^XA/^XZ**；前端在送打前同步校验并改进错误文案。货架打印增加 **「系统打印对话框（HTML）」** 备选（与销售单同源 `printHtmlDocument`）。PDA 装箱 **打印箱贴** 失败时补充说明需在接标签机的桌面端执行。

## 桌面端（Electron）

- **`desktop/lib/localPrint.js`**
  - Windows：RAW 前枚举打印机名，**名称不区分大小写**匹配；失败时列出本机已安装名称（节选）。
  - macOS/Linux：`lpstat -p` 校验 CUPS 队列；失败时附带 `lpstat -a` 节选。
  - `Get-Printer` 不可用时记录警告并跳过枚举（兼容旧环境）。
  - 送 RAW 前要求 ZPL 同时包含 **`^XA`** 与 **`^XZ`**。
  - `lp` / PowerShell 失败信息整理为可读中文提示。

## 前端

- **`desktopLocalPrint.ts`**：`validateZplForLocalPrint`；未配置打印机名称时的引导文案；`complete-local` 前对 content `trim`；未知异常时的兜底提示。
- **`printRackLabelHtml.ts`**（新）：货架条码 **Code128 + 信息行**，走 **系统打印对话框**；不入队、不核销。
- **`pages/racks/index.tsx`**：打印改为下拉 — **本机标签机（ZPL 入队）** / **系统打印对话框（HTML）**；ZPL 失败 toast 提示可改用 HTML。
- **`pages/pda/pack.tsx`**：本机打印失败时说明 PDA 仅提交任务、需在桌面端出纸。

## 升级说明

1. **桌面**：重新打包或拉取 CI 生成的安装包；标签 RAW 仍须 **ERP 打印机名与系统队列名一致**。
2. **后端**：本版本 **无强制数据库变更**（沿用 v0.3.19 API）。

## 相对 v0.3.19

ZPL 本机打印前置校验与错误提示；货架 HTML 系统打印退路；PDA 箱贴失败说明。
