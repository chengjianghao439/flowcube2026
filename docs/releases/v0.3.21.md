# FlowCube v0.3.21

## 摘要

聚焦 **ZPL 本机 RAW 打印稳定性** 与 **打印机管理**：后端 **`printerName` 仅使用 `printers.name`**（不再用 `code` 冒充系统队列名）；Electron **`print-zpl`** 用 **`getPrintersAsync`** 与「从本机添加」同源解析规范队列名；**`localPrint`** 去掉易误判的 PowerShell 预校验并支持 UTF-8 BOM 剥离。**打印机管理** 增加名称规范化、列表过滤、刷新后清空无效选中、提交时用 **`pickSystemPrinterRow`** 写入系统名；后端 **`printers.code` 全局唯一** 自动加后缀。货架标签去掉 **HTML 系统打印** 退路，仅保留 ZPL 流程。

## 后端

- **`print-jobs.service.js`**：`fmt.printerName` 仅来自 **`printer_name`**（去空格），为空则为 `null`。
- **`printers.service.js`**：创建/更新时 **名称规范化**（NFC、空白符）；**`allocateUniqueCodeGlobally`** 避免 `code` 表级唯一与其它租户冲突。

## 桌面端（Electron）

- **`main.js`**：`resolveCanonicalPrinterNameForRaw`（`getPrintersAsync` + 规范化 + `displayName` 匹配）后送 RAW。
- **`lib/localPrint.js`**：移除 **Get-Printer / lpstat** 前置拦截；保留 **^XA/^XZ** 校验；ZPL **去 BOM**。

## 前端

- **`utils/printerName.ts`**：`normalizeSystemPrinterName`、`pickSystemPrinterRow`。
- **`settings/printers`**：本机列表过滤、规范化「已在 ERP」判断、确认添加用解析后的 **`picked.name`**；刷新后修正选中项。
- **`pages/racks`**：删除 **`printRackLabelHtml`**，货架仅 **ZPL 入队 + 本机打印**。

## 升级说明

1. **后端**：部署并重启。
2. **桌面**：安装 CI 构建的新包；若旧数据 **`printers.name` 为空**，请在 **打印机管理** 用「从本机添加」补全或与系统队列名一致。

## 相对 v0.3.20

ZPL 队列名与 Chromium 枚举对齐；打印机添加与全局编码冲突处理；移除货架 HTML 标签打印。
