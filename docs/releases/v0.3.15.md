# FlowCube v0.3.15

## 摘要

移除独立 **print-client**（脚本与 CI），打印改为 **FlowCube 桌面端本机直连 ZPL**；登录/退出时 **重置 ERP 标签页** 仅保留仪表盘；后端精简 SSE/轮询/工作站注册等与旧客户端相关的接口。

## 后端

- **移除**：`GET /api/print-jobs/listen/station`、`GET /api/print-jobs/listen/:printerCode`、`GET /api/print-jobs/claim-next`；`POST /api/printers/register-client`、`POST /api/printers/heartbeat`。
- **新增**：`POST /api/print-jobs/:id/complete-local`（普通登录用户即可核销仍为「待打印」且未被领取的任务，无需 `print:client`）。
- **货架打印**：请求头 `X-Flowcube-Desktop-Local-Print: 1` 时，响应附带 ZPL `content` / `contentType`，供桌面端本机出纸后核销。
- **`getDispatchHintForJob` / `pushToClients`**：与独立打印客户端解耦后的提示与空推送逻辑。
- **保留**：`POST /print-jobs/:id/complete`、`/:id/fail`（需 `print:client`，供自动化或外部集成）；`print_clients` 与在线客户端列表接口仍可查历史数据。

## 桌面端（Electron）

- **`desktop/lib/localPrint.js`**：主进程 ZPL → 网口 TCP（默认 9100）或 macOS/Linux **CUPS raw**（`lp -o raw`）。
- **`preload` / `main`**：暴露 **`flowcubeDesktop.printZpl`**。

## 前端

- **`desktopLocalPrint.ts`**：本机出纸 + 调用 `complete-local`；**打印机管理** 增加「本机直连出纸（ZPL）」配置（`localStorage`）。
- **货架 / PDA 箱贴**：桌面端在配置 ZPL 目标后自动本机打印并核销队列。
- **`useLogin` / `performSessionLogout`**：登录成功或退出会话时 **`closeAll()`** 工作区标签，重新进入后仅 **仪表盘**。

## 仓库与 CI

- **删除**：`frontend/print-client.js`、`frontend/print-client.cjs`、`frontend/config.json`、**`.github/workflows/build-print-client.yml`**。

## 升级说明

1. **后端**：部署并 **重启**。
2. **桌面**：重新 **`npm run dist:win`**（或 CI）打包分发；用户在 **设置 → 打印机管理** 填写 **ZPL 网口 IP**（Windows）或 **CUPS 队列**（macOS/Linux）。
3. **不再提供** 独立 print-client 可执行文件与对应 Workflow；出纸依赖 **FlowCube 桌面端** 或自建调用 **`complete`/`fail`** 的集成。

## 相对 v0.3.14

移除旧打印客户端全链路，新增桌面本机直连与队列核销；ERP 多标签持久化在登录/登出时重置为仅仪表盘。
