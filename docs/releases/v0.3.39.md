# v0.3.39

## 桌面端

- **API 根地址**：若用户已在设置或登录流程中保存过 `API_BASE_URL`，启动探测与 Axios 网络回退**不再**自动切换到其它可达主机（例如本机 `localhost:3000`），避免连到错误/空库后出现列表无数据、用户信息异常等问题。
- **自助恢复**：若历史版本曾写错地址，请使用 **Ctrl+Shift+S** 核对并保存正确后端根地址（无 `/api`），然后重新登录。

## 后端

- **`app.set('json charset', 'utf-8')`**：统一声明 JSON 为 UTF-8，减少客户端按错误本地代码页解析中文的情况。

## 发布与部署

1. 仓库推送 tag **`v0.3.39`**（与 `desktop/package.json` 的 `version` 一致）后，Actions **Build Desktop Installer** 将构建 Windows 安装包并创建/更新 GitHub Release。
2. 若已配置 Secrets：`SSH_PRIVATE_KEY`、`SERVER_HOST`、`SERVER_USER`、`SERVER_DOWNLOADS_PATH`（及可选 `SERVER_SSH_PORT`），工作流会将 `desktop/release/*.exe` **上传到服务器** `downloads` 目录。
3. 部署后端时请将仓库内 **`backend/downloads/latest.json`** 与上述 **exe 文件名** 一并同步到线上一致，以便同域 `/downloads/` 在线更新与审计下载一致。

## 审计建议（Windows）

- 安装后控制台/日志中 **`🔥 BUILD VERSION:`** 应与 **0.3.39** 一致。
- 确认 **`GET https://<你的API域名>/downloads/latest.json`** 中 `version`、`filename` 与本发行一致，且 `/downloads/<filename>` 可下载同一构建的安装包。
