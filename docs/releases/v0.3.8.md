# FlowCube ERP v0.3.8 更新说明

对应 Git 标签：`v0.3.8`（与 `desktop/package.json`、`frontend/package.json` 版本一致）。

## 更新内容（本版）

### 货架条码（RCK）

- 数据库：`warehouse_racks.barcode` 唯一，存量回填 `RCK` + 6 位数字；新建货架写入后自动生成条码。
- 打印：`buildRackLabelZpl` + `enqueueRackLabelJob`；`POST /api/racks/:id/print-label`。
- 前端：解析 `RCKxxxxxx`；新页 **货架管理**（`/racks`）；货架表单展示条码与「打印货架标」。

### 同仓容器拆分（散件）

- 引擎：`SOURCE_TYPE.container_split`，`splitContainer`（ACTIVE、未锁定、`0 < qty < remaining_qty`；新容器继承库位/批次，`parent_id` 指向源容器）。
- API：`POST /api/inventory/containers/:id/split`，body `{ qty, remark?, printLabel? }`；可选入队新容器标。
- PDA：`/pda/split`（权限 `pda:split`，主管/拣货/打包）；扫码 CNT → 数量 → 提交。

### 物流箱贴（BOX）

- 打印：`buildPackageLabelZpl` + `enqueuePackageLabelJob`（任务号、客户、行数/件数摘要）。
- `PUT /api/packages/:id/finish` 成功后自动尝试入队箱贴；`POST /api/packages/:id/print-label` 补打。
- PDA 打包：每箱 **打印箱贴**。

### 数据库迁移

- 部署前执行：`backend/src/database/051_warehouse_racks_barcode.sql`。

## 自动更新与桌面端

1. 安装低于 **0.3.8** 的旧版桌面端后，确保后端 `GET /downloads/latest.json` 返回本版本 `version` 与 `filename`。
2. 客户端检查更新应提示 **0.3.8**，安装包名与 CI 一致：`FlowCube-ERP-Setup-0.3.8.exe`。
3. **Build Desktop Installer**：推送标签 `v0.3.8` 触发 GitHub Actions 构建 Windows 安装包并发布 Release。
