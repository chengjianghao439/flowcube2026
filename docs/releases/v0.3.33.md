# FlowCube v0.3.33

## 摘要

**TSPL 中文与佳博/TSC**：队列有作业但不出纸常见于 **UTF-8 中文 `TEXT` 与固件默认代码页不一致**。本版在生成的 TSPL 中统一加入 **`CODEPAGE 936`**，且在 **Windows 桌面端**将整段 TSPL 以 **GB18030** 编码再送 RAW（依赖 **`iconv-lite`**）；脚本若声明 UTF-8 代码页或设置 **`FLOWCUBE_TSPL_BYTES=utf8`** 则仍送 UTF-8。

## 主要变更

- `backend/src/modules/print-jobs/labelTsplTemplate.js`：`tsplHeaderLines` 含 `CODEPAGE 936`；自定义 TSPL body 无 `CODEPAGE` 时在 `CLS` 前自动插入。
- `desktop/lib/localPrint.js`：Win32 TSPL 默认 `gb18030`；与脚本内 UTF `CODEPAGE` 联动；日志 `enc=utf8|gb18030`。
- `desktop/package.json`：新增依赖 `iconv-lite`。
- `docs/RELEASE.md`：TSPL 中文编码说明。

## 升级说明

- **后端**：需部署以生成带 `CODEPAGE 936` 的 TSPL（旧任务内容无此项时，请重新触发打印）。
- **桌面端**：须安装本版（含 `iconv-lite` 与新的 RAW 编码逻辑）。
- **`GET /downloads/latest.json`**：与仓库 `backend/downloads/latest.json` 对齐。

## 相对 v0.3.32

专门改善佳博等 **TSPL + 中文** 本机 RAW「有队列、提示成功但不出纸」场景。
