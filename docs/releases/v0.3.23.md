# FlowCube v0.3.23

## 摘要

**打印机管理**：区分 **「设备类型」**（添加打印机时的硬件分类）与 **「绑定用途」**（业务场景），避免与「面单」等字面重复造成误解。**绑定用途** 支持 **再次点击已绑定项以解除绑定**（调用既有 `DELETE /api/printer-bindings/:type`）。用途列表扩展为 **电子面单、商品标签、库存标签、货架标签、散件容器标签**；调度上 **`rack_label` / `container_label`** 可单独绑定打印机，**未配置时自动回退到「库存标签」绑定**，兼容仅绑定库存机的旧配置。

## 后端

- **`printer-bindings.routes.js`**：`VALID_TYPES` 增加 `rack_label`、`container_label`；`DELETE` 校验类型。
- **`print-dispatch.js`**：`normalizeJobType` 区分货架/容器任务键；`bindingFallbackChain` + `fetchBindingCandidates` 实现专用绑定缺失时回退 `inventory_label`。
- **`print-jobs.service.js`**：货架标任务 `rack_label`，容器标任务 `container_label`。

## 前端

- **`settings/printers`**：添加对话框「设备类型」说明；绑定弹窗 5 类用途 + 文案；绑定/解绑切换与加载态。

## 升级说明

1. **后端**：部署并重启。
2. **桌面**：安装 CI 构建的新安装包（见 GitHub Release）；前端随桌面包更新。
3. **`/downloads/latest.json`**：已与仓库示例同步时可指向本版本 `version` / `filename`。

## 相对 v0.3.22

打印机用途绑定可解除；货架/散件容器独立绑定键与回退逻辑；设备类型与用途文案澄清。
