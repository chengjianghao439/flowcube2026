# v0.3.97

## 退货模块 PDA 化

### 数量校验修复
- 采购退货可退数量改为基于实际收货量，防止未收货商品被退货
- 销售退货可退数量改为基于实际发货量，防止未发货商品被退货

### 采购退货 PDA 出库
- 确认采购退货单后自动创建仓库拣货任务（task_type=purchase_return）
- 复用现有 PDA 拣货→出库流程，退货商品走 FIFO 扣减
- 出库完成后自动同步退货单状态并冲减应付账款

### 销售退货 PDA 入库
- 新建 return_tasks 表，支持收货→质检→上架三阶段 PDA 流程
- 确认销售退货单后自动创建退货 PDA 任务
- PDA 收货：扫描退回商品，创建容器（PENDING_QA 待质检状态）
- PDA 质检：确认退回数量，容器转为 PENDING_PUTAWAY 待上架
- PDA 上架：扫描容器+库位，容器转为 ACTIVE 可用库存
- 全部上架完成后自动同步退货单状态并冲减应收账款

### 仓库任务取消增强
- 取消任务时清理关联包裹（packages.status→3）和包裹标签打印任务
- 取消事件记录每个锁定容器的库位信息，供操作员物理归还参考
- 修复 PENDING→CANCELLED 状态转换缺失

### 数据库迁移
- 083: packages.status 注释更新（新增状态 3=已取消）
- 084: warehouse_tasks 新增 task_type、return_id 字段
- 085: 新建 return_tasks/return_task_items 表，容器新增 PENDING_QA 状态
