# FlowCube v0.3.80

极序 Flow v0.3.80 更新说明

发布时间：2026-05-03

本次更新重点提升了 PDA 拣货、仓库任务、打印任务和页面易读性。

## 用户可见更新说明

主要更新：

1. 优化 PDA 拣货扫码校验，减少扫错容器、扫错仓库带来的异常。
2. 优化仓库任务和波次状态保护，降低重复点击和并发操作导致的状态异常。
3. 优化打印任务状态处理，提升打印失败、重试、完成回执的可靠性。
4. 增加 PDA 设备会话识别能力，为后续 PDA 设备管理做准备。
5. 操作日志、异常工作台、打印查询、PDA 错误提示等页面改为更容易理解的中文说明。
6. 废弃旧版手工拣货 / 复核接口，统一使用 PDA 扫码流程。

使用提示：

- 浏览器端请刷新页面后使用。
- PDA 端如页面异常，请退出后重新进入；如有 APK 更新提示，请按提示更新。
- 桌面端会自动检测更新，如未自动更新可重新打开应用。
- 本次更新通常不需要重新登录；如果遇到登录状态异常，请重新登录。

## 技术更新说明

- PDA 拣货扫码改为核对真实库存容器，降低条码与任务明细误匹配风险。
- `lockContainer()` 增加二次 SQL 守卫，避免并发场景下容器锁定被覆盖。
- `picking_waves` 状态更新增加 CAS 保护，降低波次重复推进风险。
- 旧仓库任务手工拣货 / 复核接口返回 `410`，引导使用 PDA 扫码流程。
- `updatePriority()` 增加终态保护，已完成、取消等终态任务不再允许调整优先级。
- `print_jobs` 的 `complete` / `fail` / `retry` 增加状态 CAS，降低重复回执和并发重试风险。
- PDA device session 第一阶段保持 optional，不阻断旧 PDA 设备作业。
- 新增 `pda_devices` / `pda_device_sessions` 数据库迁移，为后续设备管理做准备。
- 操作日志页面完成白话化，列表默认隐藏 HTTP 方法、接口路径、状态码等原始技术信息，详情中保留原始字段。
- 前端专业术语白话化第二阶段覆盖异常工作台、打印查询、入库打印记录、库存总览、打印机设置、通用错误状态、全局 API 错误提示、PDA 错误提示和 403 页面。

## 测试环境回归结果

- 前端 Electron/Web 构建需通过 `npm --prefix frontend run build`。
- PDA 前端构建需通过 `npm --prefix frontend run build:pda`。
- 后端构建需通过 `npm --prefix backend run build`。
- GitHub Actions 会分别验证浏览器部署、PDA APK 构建与发布、Windows 桌面安装包构建与自动更新 manifest。

## 已知注意事项

- 管理员排查所需的原始接口路径、错误码、打印机编号、PDA 错误原文仍保留在详情、tooltip、小字或错误详情中。
- 桌面端正式 Windows 安装包由 GitHub Actions 在 Windows 环境构建，并通过 `/latest.json` 与 `/versions/` 发布。
- PDA APK 版本以 `backend/apk/version.json` 与 Android `versionCode/versionName` 为准，发布时必须保持一致。
