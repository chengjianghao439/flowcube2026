# v0.3.98

## 安全加固

### 密码日志脱敏（致命）
- 操作日志中间件新增 `sanitizeBody()` 函数，对 `password`、`token`、`secret` 等 10 个敏感字段脱敏
- 修复登录/修改密码等接口将密码明文写入 `operation_logs` 表的问题

### 文件上传白名单
- 导入模块 `multer` 配置新增 `fileFilter`，仅允许 `.xlsx`、`.xls`、`.csv` 三种 MIME 类型
- 防止非 Excel 文件上传触发 `xlsx` 库解析漏洞

### 其他安全改进
- `auth.routes.js`：移除运行时 `require()`，统一使用共享 `validateBody`，登录保持 `safeParse` 特定错误格式
- `deploy.example.env` 重命名为 `deploy.env.example`，避免安全扫描 `*.env` 模式误报
- `PdaErrorBoundary`：移除冗余的 `window.location.href` 赋值

## 性能优化

### N+1 查询消除
- 容器追溯：新增 `resolveSourceDocumentsBatch()` 批量解析函数，按来源类型分组查询替代逐行 DB 调用
- 库存一致性校验：`checkStockConsistency` 使用 `LEFT JOIN` 替代逐行 `SELECT`，单次查询完成

### 启动性能
- `JWT_SECRET` 从 getter 改为模块加载时一次性缓存，避免每次签名/验证重复读取环境变量

## 数据库改进

### 新增索引
- `purchase_order_items`、`sale_order_items`：新增 `idx_order_id`，解决订单详情全表扫描
- `inventory_check_items`：新增 `idx_check_id` + `uk_check_product` 唯一约束
- `transfer_order_items`、`package_items`：新增 `idx_product_id`
- `picking_wave_tasks`：新增 `uk_wave_task` 唯一约束
- `picking_wave_items`：新增 `uk_wave_product` 唯一约束
- `operation_requests`：补充 `utf8mb4` 字符集声明

### 流水号原子递增
- 新建 `daily_sequences` 表，`generateDailyCode` 改用 `UPDATE + LAST_INSERT_ID()` 原子操作
- 消除并发场景下 `SELECT COUNT(*) + 1` 的竞态条件
- Pool 场景自动获取专用连接，保证 LAST_INSERT_ID 连接一致性

### 自动清理
- `operation_requests` 表新增 TTL 定时清理（每 6 小时清理超过 7 天的记录），防止无限增长

## 事务保护

### 新增事务包装
- `deleteOrder`：事务内 `SELECT ... FOR UPDATE` 锁定订单行，替代跨连接的读写分离
- `assignContainerLocation`：容器-库位绑定操作添加事务 + `FOR UPDATE` 锁

### 连接池改进
- 连接池大小支持环境变量 `DB_POOL_SIZE` 配置（默认 10）
- 新增 `connectTimeout` / `acquireTimeout` 各 10 秒，防止连接池耗尽无响应

## 前端改进

### Tab 级错误边界
- 新建 `TabErrorBoundary` 组件，包装 `KeepAliveOutlet`
- 单个 Tab 渲染崩溃不再影响全局，其他 Tab 正常工作

### Bug 修复
- `useDirtyGuard` 参数传递修正：函数误传为 `tabPath`，商品表单未保存保护现已生效

### 操作日志完善
- `MODULE_MAP` 从 13 个 API 前缀扩展到 29 个，覆盖全部业务模块

## 冗余清理
- 移除 `migrate.js` 中重复的 `warehouse_racks` CREATE TABLE 语句
- 移除 `containerEngine.js` 中 `INBOUND_TASK` 的冗余 ACTIVE 状态校验
