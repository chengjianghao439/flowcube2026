## 一、PDA 端（问题最集中，直接影响一线操作员效率）

### 1.1 过度技术化的提示文本

| 原文 | 问题 | 建议修改 |
|------|------|----------|
| `后端未授予收货、拣货、分拣……等 PDA 权限` | "后端"和"授予"是技术术语 | `当前账号没有PDA操作权限，请联系管理员` |
| `当前账号没有收到后端返回的权限信息` | 整句是调试日志风格 | `未获取到权限信息，请重新登录` |
| `因此不会放开任何 PDA 作业入口` | "放开入口"读起来很怪 | `PDA 功能已受限` |
| `任务已入队，但响应中缺少 ZPL 或任务 ID，本机未送 RAW` | 全句是技术 debug 输出 | `打印任务已提交，但未收到打印数据。请检查电脑上的极序 Flow 桌面端是否运行。` |
| `服务端仍未确认结果` | "服务端"不是用户语言 | `结果还未返回，请稍后再查` |
| `当前页面没有有效任务号` | "有效"带有编程味道 | `未找到任务，请返回列表重新选择` |
| `该商品不属于当前任务` | "属于"太正式 | `该商品不在此任务中` |

| 原文 | 问题 |
|------|------|
| `当前没有可展示的人员效率数据` | "没有可展示的" 啰嗦 |
| `当前没有可展示的异常日志` | |
| `当前没有可展示的操作员错误记录` | |
| `当前没有可展示的条码问题记录` | |
| `当前没有可展示的错误原因分布` | |
| `当前没有可展示的撤销统计` | |
| `当前时段没有可展示的扫码趋势` | |
| `当前时段没有可展示的流程堆积` | |

→ 统一改为 `暂无XX数据` 即可

### 1.3 直接英译的短语

| 原文 | 问题 | 建议 |
|------|------|------|
| `扫码上架` | 直译 "Scan & Putaway" | `扫描上架`（更通顺）|
| `当前不可提交` | Not submittable | `当前无法提交` |
| `数量无效` | Invalid quantity | `数量不正确` |
| `服务端仍未确认结果` | Server hasn't confirmed | `正在确认结果，请稍候` |
| `未绑定客户端` | Client not bound | `未连接桌面端` |
| `收口中…` | Closing (直译) | `收口完成中…` |
| `分拣格` | Sort bin (直译 "格") | `分拣格` 可保留，但界面中可加注"货道" |

## 二、ERP 管理端

### 2.1 功能描述生硬

| 原文 | 位置 | 问题 | 建议 |
|------|------|------|------|
| `一眼看出今天哪个时段最忙` | 仪表盘 | 想口语化但拗口 | `查看今日各时段繁忙程度` |
| `便于针对性修正` | 报表 | 像翻译腔 | `方便定位问题并修正` |
| `优先基于原单带价` | 采购 | 过于精简，看不懂 | `优先沿用原订单价格` |
| `可集中收口` | 审批页 | "收口"太技术 | `可在统一入口处理` |
| `任务完成分拣后进入此列表` | 分拣页 | 被动语态 | `已分拣完成的任务` |
| `后台确认订单后将在此显示` | 销售页 | 主语模糊 | `订单确认后会自动显示在这里` |
| `集统一收口收货、出库、库存和管理异常` | 工作台 | 句子结构英文化 | `统一处理收货、出库、库存和异常` |
| `默认按最近时间范围打开` | 报表 | 像提示工程输出 | `默认显示最近数据` |
| `保留轻 BI 方式查看` | 报表 | "轻BI"突兀 | `以报表形式查看` |

### 2.2 表头/标签优化

| 原文 | 建议 |
|------|------|
| `原始接口路径` → `接口路径` |
| `原始操作人` → `操作人` |
| `原始状态码` → `状态码` |
| `原始模块名` → `模块` |
| `可绑定商品` → `可关联商品` |
| `操作时间线` → `操作记录` |
| `包裹总数` → `总包裹数` |

### 2.3 错误提示/空状态

| 原文 | 建议 |
|------|------|
| `当前条件下没有可展示的内容` | `当前条件下没有数据` |
| `没有符合条件的记录，请尝试调整筛选条件` | `没有匹配的记录，试试调整筛选条件` |
| `当前列表没有任何记录` | `暂无记录` |
| `该目录下暂无内容` | `暂无内容` |
| `请求已发出，请保持当前页面并等待结果返回` | `请求已提交，请等待处理完成` |

## 三、后端 API 消息

后端消息相对简短标准，但存在几处可优化：

| 原文 | 位置 | 建议 |
|------|------|------|

| `出库任务已创建，等待仓库执行` | sale.controller | `出库任务已创建，等待仓库操作` |

| `PDA 不允许取消任务，请在 ERP 后台操作` | pda routes | `请在电脑端 ERP 中取消任务` |

## 四、建议修改优先级

| 优先级 | 范围 | 原因 |
|--------|------|------|
| P0 - 立即 | PDA 错误提示（部分） | 一线操作员每天高频看到，生硬文案降低效率 |
| P1 - 本周 | PDA 空状态 + 功能描述 | 影响操作流畅度 |
| P2 - 本月 | ERP 描述文本 + 后端消息 | 影响不大，统一风格即可 |
| P3 - 低优 | 标准 CRUD 消息（"查询成功"等） | 功能无误，中文任何说法都能理解 |
