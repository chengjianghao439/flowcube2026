### □ P0-2：Docker Compose 加固（5 分钟）

修改 `docker-compose.yml` — MySQL 端口改为仅本机绑定：

```diff
-      - "${MYSQL_PORT:-3306}:3306"
+      - "127.0.0.1:${MYSQL_PORT:-3306}:3306"
```

后端端口同样加固：

```diff
-      - "3000:3000"
+      - "127.0.0.1:3000:3000"
```

生产环境使用 `docker-compose.prod.yml` 覆盖完全关闭 MySQL 端口暴露：

```bash
# 部署时执行
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d --build
```

### □ P0-3：npm audit 漏洞修复（15 分钟）

**后端**：

```bash
cd backend
```

**前端**：


## P1 — 本周执行（代码质量，约 25 分钟）

### □ P1-1：调试代码清理（10 分钟）

```bash
# 第 1 步：预览扫描（只报告，不改文件）
bash scripts/cleanup-debug-logs.sh

# 审查生成的 .debug-cleanup-summary-*.txt 文件
cat .debug-cleanup-summary-*.txt

# 第 2 步：确认无误后执行清理

```

**清理规则说明**：脚本只删除以下内容：
- 单独成行的 `debugger` 语句
- 单独成行的 `console.log(...)` 调用（不影响 `logger.info()` 等业务日志）
- 不删除含有 `return`、赋值等复合逻辑的行

### □ P1-3：版本号对齐确认（2 分钟）

```bash
node -e "
const pkgs = {
  backend: require('./backend/package.json'),
  frontend: require('./frontend/package.json'),
  desktop: require('./desktop/package.json')
};
for (const [name, pkg] of Object.entries(pkgs)) {
  console.log(name + ': ' + pkg.version);

```

**红线**：三个版本号必须一致，否则发布时 Docker 构建会回退（详见 skill 中的 Docker 版本号回退问题）。

### □ P1-4：死代码标记（8 分钟）

**后端未引用文件标记**：

```bash
# 检查未引用的 controller
for f in $(find backend/src/modules -name '*.controller.js'); do
  name=$(basename "$f" .js)
  count=$(grep -r "$name" backend/src/modules --include='*.routes.js' -l 2>/dev/null | wc -l)
  if [ "$count" -eq 0 ]; then
    echo "⚠️  可能未引用: $f"
    # 安全方式：添加 @deprecated 注释，不删除文件
    if ! grep -q '@deprecated' "$f" 2>/dev/null; then
      sed -i '1s/^/\/\/ @deprecated - 此文件未被任何路由引用，确认无动态引用后可删除\n/' "$f"
    fi
  fi
done
```

**前端未注册路由页面**：

```bash
# 检查页面目录是否都在 router 中注册
for dir in frontend/src/pages/*/; do
  name=$(basename "$dir")
  count=$(grep -r "$name" frontend/src/router --include='*.tsx' -l 2>/dev/null | wc -l)
  if [ "$count" -eq 0 ]; then
    echo "⚠️  $name — 未在 router 中注册"
  fi
done
```

### □ P2-2：Git 历史瘦身（10 分钟，仅在必要时执行）

```bash
# 检查历史中大文件
git rev-list --objects --all | \
  git cat-file --batch-check='%(objecttype) %(objectname) %(objectsize) %(rest)' | \
  awk '/^blob/ {print $3, $4}' | sort -rn | head -20

# 如有大文件，使用 BFG 清理（Docker 方式）
# ⚠️ 该操作会重写 Git 历史，需与所有协作者同步
docker run -v $PWD:/repo \
  ghcr.io/rtyley/bfg-repo-cleaner:latest \
  --delete-files 'flowcube_backup.sql' --delete-files '*.apk' --delete-files '*.exe' \
  /repo

git reflog expire --expire=now --all && git gc --prune=now --aggressive
```

## 回滚方案

### 如果 Docker 部署后出问题

```bash
# 从备份恢复
cp ~/flowcube-backup-$(date +%Y%m%d)/docker-compose.yml docker-compose.yml

# 回到上一个提交
git log --oneline -5
git reset --hard <上一个稳定提交>

# 重新部署
ssh flowcube-prod 'cd /opt/flowcube && git pull && docker compose up -d --build'
```

### 如果依赖升级导致兼容问题

```bash
# 回滚 package.json 和 lock 文件
git checkout HEAD~1 -- backend/package.json backend/package-lock.json
git checkout HEAD~1 -- frontend/package.json frontend/package-lock.json

# 重新安装
npm ci --prefix backend
npm ci --prefix frontend
```

### 如果 Gitleaks 误报

在项目根目录添加 `.gitleaks.toml`：

```toml
# .gitleaks.toml
title = "FlowCube Gitleaks Config"

[allowlist]
# 排除测试用的假凭据
paths = [
  "tests/seedTestData.js",
  "tests/resetBusinessData.js",
  "deploy/production.example.json"
]
```
