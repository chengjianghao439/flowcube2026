# 换服务器与桌面端自动更新说明

> 面向所有人（不只技术）。如果你在纠结「换服务器 / 改服务器地址，桌面端会不会用不了、怎么平稳换过去」，看这一份就够了。

---

## 一、先记住这一句

**桌面端（Windows 安装包）连哪个服务器，是「焊死」在安装包里的，装好后用户改不了。**

也就是说：

- 服务器地址不是用户在登录页填的，而是**打包那一刻**就写进安装包的（来自 GitHub Variable `VITE_ERP_PRODUCTION_ORIGIN`）。
- 用户那边没有「改服务器地址」的入口（旧版的 `Ctrl+Shift+S` 快捷键已彻底移除）。
- 所以**要换服务器地址 = 必须重新打包、重新发一版桌面端**，没有别的办法。

> 浏览器端不受影响：浏览器访问哪个网址，就连哪个服务器，换地址只是换个网址打开而已。

---

## 二、为什么要这样设计

早期允许用户本地保存/手动改地址，结果踩了坑：**老机器 localStorage 里残留的旧地址，会盖过新版安装包里的新地址**，导致「明明发了新版连新服务器，老电脑却还连老地址、看到空库或错数据」，而且还没有快捷键能纠正。

现在的规则简单粗暴、不会错：

> **桌面端只认安装包内置的地址，忽略本地一切历史残留。改地址 = 重新发版，一次性全量生效。**

---

## 三、三个最常被问到的问题

### Q1：换了服务器地址，已经装了旧桌面端的用户会不会直接用不了？

**会暂时连不上，但不会「坏」。** 旧桌面端还在连旧地址：

- 如果**旧服务器还活着** → 旧桌面端继续正常用，感知不到变化。
- 如果**旧服务器关了 / 旧地址不通** → 旧桌面端会停在「无法连接服务器」的提示页，不会白屏、不会丢数据。

让用户**重新下载安装一次新版桌面端**，新版连新地址，就恢复了。

### Q2：那能不能像平时一样让它「自动更新」过去，不用手动重装？

要分情况：

- **自动更新依赖「能连上服务器」**：桌面端是靠连当前内置地址去检查 `/latest.json` 才知道有没有新版。
- 只要**旧地址此刻还能连通**（旧服务器没关、或新旧地址都指向同一台机/同一个可达入口），推一版新桌面端，旧客户端就能自动检测到并更新到「连新地址」的版本，用户无需手动重装。
- 但如果**旧地址已经彻底不通**，自动更新这条路也断了（它连检查更新的请求都发不出去），**只能让用户手动下载一次新版**。

> 结论：换地址这种事，**别等旧地址关了再发新版**。趁旧地址还通的时候先发好新版让客户端自动更新；实在赶不上的，就走「手动装一次」。

### Q3：为什么强烈建议用「域名」而不是「IP」？

因为**用域名以后再换服务器，就不用动桌面端了**：

| | 内置 **IP**（如 `https://47.93.228.251`） | 内置 **域名**（如 `https://erp.你的公司.com`） |
|---|---|---|
| 换服务器（换机器/换 IP） | ❌ 必须重新打包发版 + 客户端更新/重装 | ✅ 只改 DNS 解析指向新机器，桌面端**无需任何改动** |
| 用户感知 | 可能要手动重装一次 | 几乎无感 |
| HTTPS 证书 | 只能自签名（IP 没法签正规证书），客户端要信任改造 | 可以用正规证书（Let's Encrypt 等），干净 |

一句话：**IP 是把地址焊死在每一台客户端上；域名是把「焊死」收敛到一处 DNS，换服务器只动 DNS。**

## 四、实操：要换服务器地址 / 换服务器，怎么平稳过去

> 适用：把桌面端内置地址从旧地址换成新地址（IP→IP、IP→域名、或换新服务器）。

1. **先把新服务器准备好**：新地址（域名或 IP）能正常访问、后端健康检查 `/api/health` 返回 ok、HTTPS 证书有效、数据已迁移到位。
2. **改打包注入的地址**：把 GitHub Variable `VITE_ERP_PRODUCTION_ORIGIN` 改成新地址（建议域名，带 `https://`，不要带结尾 `/`）。
   - 本地打包测试时对应改 `deploy/production.local.json` 的 `erpOrigin`。
3. **趁旧地址还通**，正常发一版桌面端（走 [发版流程](DEPLOY.md) / `release-flowcube` 技能，升版本号 + 打 `v*` tag）。
   - 这样在线的旧客户端会**自动检测到新版并更新**，更新后即连新地址。
4. **通知用户**：自动更新没覆盖到的（比如当时没开、或旧地址已不通的），让他们到下载页**手动装一次新版**。
5. 确认大家都切到新地址后，再考虑下线旧地址 / 旧服务器。

> ⚠️ 不要顺序搞反：**先关旧服务器、再发连新地址的版** = 旧客户端既连不上旧地址、又收不到更新通知，只能挨个手动重装。

---

## 五、和「域名备案」的关系

域名 `jixuflow.com` 已购、备案推进中。备案通过、域名可正常服务后：

1. 按上面第四节，把内置地址从 `https://47.93.228.251` 换成域名；
2. 换正规 HTTPS 证书，撤销当前为了「IP + 自签名证书」做的客户端信任改造；
3. 此后再换服务器只需改 DNS，桌面端一劳永逸。
