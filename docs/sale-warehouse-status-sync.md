# 销售单状态与仓库任务状态同步策略

## 结论

销售单状态是订单级粗粒度状态；仓库任务状态是履约作业主状态。

销售单不为 `待分拣`、`待复核`、`待打包`、`待出库` 单独扩展状态，避免与 `warehouse_tasks.status` 形成两套履约状态机。销售侧页面必须同时展示：

- `销售状态`：来自 `sale_orders.status`
- `履约状态`：来自关联 `warehouse_tasks.status`

## 当前状态映射

| 仓库任务状态 | 仓库任务含义 | 销售单状态 | 销售单含义 |
| --- | --- | --- | --- |
| `3` | 待分拣 | `3` | 拣货中 / 仓库履约中 |
| `4` | 待复核 | `3` | 拣货中 / 仓库履约中 |
| `5` | 待打包 | `3` | 拣货中 / 仓库履约中 |
| `6` | 待出库 | `3` | 拣货中 / 仓库履约中 |
| `7` | 已出库 | `4` | 已出库 |

## 同步边界

允许同步：

- 创建仓库任务时：`sale_orders.status` 从 `已占库(2)` 进入 `拣货中(3)`
- 仓库任务完成出库时：`sale_orders.status` 从 `拣货中(3)` 进入 `已出库(4)`
- 仓库任务取消时：通过销售服务主入口同步到 `已取消(5)`

不做同步：

- `待分拣`
- `待复核`
- `待打包`
- `待出库`

这些阶段只属于仓库任务状态，销售单继续保持 `拣货中(3)`。

## 实现约束

- 仓库模块不得裸写 `sale_orders.status`。
- 需要同步销售单状态时，必须调用 `sale.service.js` 暴露的同步主入口。
- 前端销售列表、销售详情必须把“销售状态”和“履约状态”分开展示。
- 当用户判断实际履约阶段时，以 `warehouse_tasks.status` 为准。
