# FlowCube ERP 部署指南

> 适用版本：v1.0 | 更新日期：2026-02

---

## 一、环境要求

| 组件 | 版本要求 |
|------|----------|
| Node.js | 20+ |
| MySQL | 8.0+ 或 9.x |
| npm | 10+ |
| 操作系统 | macOS / Linux / Windows |

---

## 二、本地开发部署（推荐新手）

### 1. 克隆或下载项目

```bash
cd ~/your-workspace
# 将 flowcube 目录放入工作目录
```

### 2. 安装 Node.js（如未安装）

```bash
# macOS：通过 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
source ~/.nvm/nvm.sh
nvm install 20
node -v   # 应显示 v20.x.x
```

### 3. 配置数据库

1. 安装并启动 MySQL 8.0+
2. 创建数据库：
```sql
CREATE DATABASE flowcube CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```
3. 配置后端环境变量：
```bash
cd flowcube/backend
cp .env.example .env
# 编辑 .env，填写以下内容：
```
```env
PORT=3000
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的MySQL密码
DB_NAME=flowcube
JWT_SECRET=请替换为随机字符串
JWT_EXPIRES_IN=7d
CORS_ORIGIN=http://localhost:5173
```

### 4. 安装依赖

```bash
# 后端
cd flowcube/backend
npm install

# 前端
cd flowcube/frontend
npm install
```

### 5. 启动服务

**后端**（首次启动会自动建表）：
```bash
cd flowcube/backend
node index.js
# 看到 "FlowCube API 运行在 http://localhost:3000" 即成功
```

**前端**（新开终端）：
```bash
cd flowcube/frontend
npm run dev
# 访问 http://localhost:5173
```

### 6. 初始账号

| 账号 | 密码 | 角色 |
|------|------|------|
| admin | admin123 | 超级管理员 |

> ⚠️ 首次登录后请立即在右上角用户菜单修改密码。

## 三、生产环境部署（Docker 一键部署）

### 前提条件

- 已安装 Docker 和 Docker Compose

### 启动步骤

```bash
cd flowcube

# 1. 从示例生成环境变量文件（推荐，避免改 compose）
cp deploy.example.env .env
# 编辑 .env：DB_PASSWORD、JWT_SECRET（须 ≥32 字符）、CORS_ORIGIN（与浏览器访问前端的地址一致，如 http://你的公网IP）
# JWT_SECRET 可用：openssl rand -hex 32

# 2. 一键启动（Compose 会自动读取根目录 .env）
docker compose up -d

# 查看运行状态
docker compose ps

# 查看后端日志
docker compose logs backend
```

**访问地址：**
- 前端：`http://服务器IP/`（Nginx 80 端口；若经 HTTPS 反代则用你的域名）
- 后端 API：经前端同机反代为 `http://服务器IP/api/`；仅调试时可直接访问容器映射的 `http://服务器IP:3000`

### 停止服务

```bash
docker compose down          # 停止（保留数据）
docker compose down -v       # 停止并删除数据（⚠️ 慎用）
```

## 四、数据备份

### 手动备份

```bash
cd flowcube
bash scripts/backup.sh
# 备份文件保存在 flowcube/backups/
```

### 自动定时备份（推荐）

```bash
# 编辑 crontab
crontab -e

# 添加以下行（每天凌晨 2 点备份）
0 2 * * * /path/to/flowcube/scripts/backup.sh >> /path/to/flowcube/backups/backup.log 2>&1
```

## 六、常见问题

### Q1：后端启动报错 "Access denied"
- 检查 `.env` 中的 `DB_HOST` 是否为 `127.0.0.1`（不要用 `localhost`）
- 确认 MySQL 用户名和密码正确

### Q2：前端页面空白
- 确认后端已启动（先启动后端，再启动前端）
- 浏览器打开 `http://localhost:3000/health`，应返回 `{"success":true}`

### Q3：首次启动建表失败
- 确认数据库 `flowcube` 已创建
- 确认 MySQL 用户有 DDL 权限（`CREATE TABLE`）

### Q4：端口冲突
```bash
# 查看占用端口
lsof -i :3000
# 结束占用进程
kill -9 <PID>
```

### Q5：数据库忘记密码（macOS）
```bash
# 停止 MySQL
sudo /usr/local/mysql/support-files/mysql.server stop

# 以跳过权限模式启动
sudo /usr/local/mysql/bin/mysqld_safe --skip-grant-tables &

# 连接并修改密码
/usr/local/mysql/bin/mysql -u root
# mysql> FLUSH PRIVILEGES;
# mysql> ALTER USER 'root'@'localhost' IDENTIFIED BY '新密码';
```

## 七、目录结构速览

```
flowcube/
├── backend/          # Node.js + Express 后端
│   ├── src/
│   │   ├── modules/  # 业务模块（按功能分组）
│   │   ├── database/ # SQL 迁移文件（自动执行）
│   │   ├── config/   # 数据库连接配置
│   │   ├── middleware/ # 认证、日志、错误处理
│   │   └── utils/    # 工具函数
│   ├── .env          # 环境变量（不提交到 Git）
│   └── index.js      # 入口文件
├── frontend/         # React + Vite 前端
│   ├── src/
│   │   ├── pages/    # 页面组件
│   │   ├── api/      # API 调用函数
│   │   ├── hooks/    # React Query hooks
│   │   ├── components/ # 共享 UI 组件
│   │   └── store/    # Zustand 状态管理
│   └── vite.config.ts
├── electron/         # 桌面端包装（可选）
├── scripts/          # 运维脚本（备份等）
├── docker/           # Docker 配置
├── docker-compose.yml
└── docs/             # 项目文档
```
